Generating CS classes from XSD
==============================================================================
Go to http://aviationweather.gov/dataserver, and run a query. The response has the url to the xsd file.
Download xsd and run xsd.exe tool to generate the cs class. You must use the namespace option, because 
they named all classes "response", so there will be a naming conflict.
Examples:
xsd.exe /classes /namespace:taf "%USERPROFILE%\Downloads\taf1_2.xsd"
xsd.exe /classes /namespace:metar "%USERPROFILE%\Downloads\metar1_2.xsd"
xsd.exe /classes /namespace:airsigmet "%USERPROFILE%\Downloads\airsigmet1_1.xsd"
xsd.exe /classes /namespace:aircraftreport "%USERPROFILE%\Downloads\aircraftreport1_0.xsd"
xsd.exe /classes /namespace:gairmet "%USERPROFILE%\Downloads\gairmet1_0.xsd"

After adding the generated CS files to the Visual Studio solution file, remove the attributes:
    [System.SerializableAttribute()]
    [System.ComponentModel.DesignerCategoryAttribute("code")]
because they are not recognized.
